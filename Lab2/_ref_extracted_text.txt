2.1.2. Попередня обробка даних
Оскільки набір даних містить обидва типи ознак – числові та категоріальні – перед навчанням моделі виконувалась попередня обробка:
– числові ознаки (age, fnlwgt, education-num, capital-gain, capital-loss, hours-per-week) – заповнення пропусків медіаною та масштабування StandardScaler;
– категоріальні ознаки (workclass, education, marital-status, occupation, relationship, race, sex, native-country) – заповнення найчастішим значенням та OneHotEncoder;
– рядки зі значеннями «?» видалялись (dropna);
– балансування вибірки: до 25 000 прикладів на кожен клас;
– розбиття 80/20 (train/test) зі стратифікацією.
Обробку реалізовано через ColumnTransformer та Pipeline, що забезпечує уніфіковану підготовку даних для всіх моделей.
import os
import numpy as np
import pandas as pd
from sklearn.compose import ColumnTransformer
from sklearn.impute import SimpleImputer
from sklearn.metrics import (
    accuracy_score,
    classification_report,
    confusion_matrix,
    f1_score,
    precision_score,
    recall_score,
)
from sklearn.model_selection import cross_val_score, train_test_split
from sklearn.pipeline import Pipeline
from sklearn.preprocessing import OneHotEncoder, StandardScaler
from sklearn.svm import LinearSVC
def load_income_data(path: str) -> pd.DataFrame:
    columns = [
        "age",
        "workclass",
        "fnlwgt",
        "education",
        "education-num",
        "marital-status",
        "occupation",
        "relationship",
        "race",
        "sex",
        "capital-gain",
        "capital-loss",
        "hours-per-week",
        "native-country",
        "income",
    ]
    df = pd.read_csv(
        path,
        header=None,
        names=columns,
        sep=r",\s*",
        engine="python",
        na_values="?",
    )
    return df.dropna().copy()
def build_preprocessor():
    numeric_features = [
        "age",
        "fnlwgt",
        "education-num",
        "capital-gain",
        "capital-loss",
        "hours-per-week",
    ]
    categorical_features = [
        "workclass",
        "education",
        "marital-status",
        "occupation",
        "relationship",
        "race",
        "sex",
        "native-country",
    ]
    numeric_transformer = Pipeline(
        steps=[
            ("imputer", SimpleImputer(strategy="median")),
            ("scaler", StandardScaler(with_mean=False)),
        ]
    )
    categorical_transformer = Pipeline(
        steps=[
            ("imputer", SimpleImputer(strategy="most_frequent")),
            ("onehot", OneHotEncoder(handle_unknown="ignore")),
        ]
    )
    preprocess = ColumnTransformer(
        transformers=[
            ("num", numeric_transformer, numeric_features),
            ("cat", categorical_transformer, categorical_features),
        ]
    )
    return preprocess
def main():
    lab2_dir = os.path.dirname(os.path.abspath(__file__))
    input_path = os.path.join(lab2_dir, "income_data.txt")
    df = load_income_data(input_path)
    max_per_class = 25_000
    df_le = df[df["income"] == "<=50K"].head(max_per_class)
    df_gt = df[df["income"] == ">50K"].head(max_per_class)
    df_balanced = (
        pd.concat([df_le, df_gt], axis=0)
        .sample(frac=1.0, random_state=42)
        .reset_index(drop=True)
    )
    X_df = df_balanced.drop(columns=["income"])
    y = df_balanced["income"]
    X_train, X_test, y_train, y_test = train_test_split(
        X_df, y, test_size=0.2, random_state=5, stratify=y
    )
    model = Pipeline(
        steps=[
            ("preprocess", build_preprocessor()),
            ("clf", LinearSVC(random_state=0)),
        ]
    )
    model.fit(X_train, y_train)
    y_pred = model.predict(X_test)
    print("=== Task 2.1: Linear SVM results ===")
    print("Accuracy:", accuracy_score(y_test, y_pred))
    print("Precision(>50K):", precision_score(y_test, y_pred, pos_label=">50K"))
    print("Recall(>50K):", recall_score(y_test, y_pred, pos_label=">50K"))
    print("F1(>50K):", f1_score(y_test, y_pred, pos_label=">50K"))
    print("F1_weighted:", f1_score(y_test, y_pred, average="weighted"))
    print("\nConfusion matrix:\n", confusion_matrix(y_test, y_pred, labels=["<=50K", ">50K"]))
    print("\nClassification report:\n", classification_report(y_test, y_pred))
    cv_f1 = cross_val_score(model, X_df, y, scoring="f1_weighted", cv=3)
    print("\nF1_weighted (CV=3):", cv_f1.mean(), "+/-", cv_f1.std())

ЗАВДАННЯ 2.2. ПОРІВНЯННЯ SVM З НЕЛІНІЙНИМИ ЯДРАМИ
У цьому завданні перевірялись SVM з нелінійними ядрами: поліноміальним (poly, degree=8), гаусовим (RBF) та сигмоїдальним (sigmoid). Порівняння проводилось на підвибірці по 2 000 прикладів на клас, оскільки kernel SVM має обчислювальну складність O(n²) і на великих вибірках навчання є надмірно тривалим.
2.2.1. Характеристика ядер
– поліноміальне ядро (poly, degree=8) – будує нелінійну межу у вигляді полінома 8-го степеня у просторі ознак;
– гаусове/RBF-ядро – найпоширеніше нелінійне ядро, розглядає подібність між прикладами через гаусову функцію;
– сигмоїдальне ядро (sigmoid) – нагадує нейронну мережу з одним прихованим шаром.

ЗАВДАННЯ 2.4. ПОРІВНЯННЯ КЛАСИФІКАТОРІВ НА CENSUS INCOME
По аналогії із завданням 2.3, ті самі шість алгоритмів порівнювались на датасеті income_data.txt. Для забезпечення порівнянності всі моделі отримували дані через однаковий ColumnTransformer (числові ознаки – StandardScaler, категоріальні – OneHotEncoder з sparse_output=False).
Порівняння виконувалось через Stratified 5-fold CV (5 фолдів замість 10 для скорочення часу обчислень на великому датасеті). Додатково для кожного алгоритму розраховувались метрики на hold-out тесті (80/20).
import os

def load_income_data(path: str) -> pd.DataFrame:
    columns = [
        "age",
        "workclass",
        "fnlwgt",
        "education",
        "education-num",
        "marital-status",
        "occupation",
        "relationship",
        "race",
        "sex",
        "capital-gain",
        "capital-loss",
        "hours-per-week",
        "native-country",
        "income",
    ]
    df = pd.read_csv(
        path,
        header=None,
        names=columns,
        sep=r",\s*",
        engine="python",
        na_values="?",
    )
    return df.dropna().copy()
def main():
    lab2_dir = os.path.dirname(os.path.abspath(__file__))
    input_path = os.path.join(lab2_dir, "income_data.txt")
    df = load_income_data(input_path)
    max_per_class = 25_000
    df_le = df[df["income"] == "<=50K"].head(max_per_class)
    df_gt = df[df["income"] == ">50K"].head(max_per_class)
    df_balanced = (
        pd.concat([df_le, df_gt], axis=0)
        .sample(frac=1.0, random_state=42)
        .reset_index(drop=True)
    )
    X_df = df_balanced.drop(columns=["income"])
    y = df_balanced["income"]
    numeric_features = [
        "age",
        "fnlwgt",
        "education-num",
        "capital-gain",
        "capital-loss",
        "hours-per-week",
    ]
    categorical_features = [c for c in X_df.columns if c not in numeric_features]
    try:
        ohe = OneHotEncoder(handle_unknown="ignore", sparse_output=False)
    except TypeError:
        ohe = OneHotEncoder(handle_unknown="ignore", sparse=False)
    preprocess_dense = ColumnTransformer(
        transformers=[
            (
                "num",
                Pipeline(
                    steps=[
                        ("imputer", SimpleImputer(strategy="median")),
                        ("scaler", StandardScaler()),
                    ]
                ),
                numeric_features,
            ),
            (
                "cat",
                Pipeline(
                    steps=[
                        ("imputer", SimpleImputer(strategy="most_frequent")),
                        ("onehot", ohe),
                    ]
                ),
                categorical_features,
            ),
        ]
    )
    models = []
    models.append(("LR", LogisticRegression(solver="liblinear", max_iter=200)))
    models.append(("LDA", LinearDiscriminantAnalysis()))
    models.append(("KNN", KNeighborsClassifier(n_neighbors=15)))
    models.append(("CART", DecisionTreeClassifier(random_state=1)))
    models.append(("NB", GaussianNB()))
    models.append(("SVM", SVC(gamma="scale")))
    kfold = StratifiedKFold(n_splits=5, random_state=1, shuffle=True)
    results = []
    names_ = []
    for name, clf in models:
        pipe = Pipeline(steps=[("preprocess", preprocess_dense), ("clf", clf)])
        scores = cross_val_score(pipe, X_df, y, cv=kfold, scoring="accuracy")
        results.append(scores)
        names_.append(name)
        print(f"{name}: {scores.mean():.4f} ({scores.std():.4f})")
    plt.figure(figsize=(8, 4))
    plt.boxplot(results, labels=names_)
    plt.title("Algorithm Comparison (income_data.txt)")
    plt.ylabel("Accuracy (CV)")
    plt.show()

2.5.1. Характеристика класифікатора
RidgeClassifier – лінійний класифікатор з L2-регуляризацією (ridge). Він перетворює задачу класифікації в регресійну та відрізняється від логістичної регресії способом обчислення функції втрат. L2-регуляризація штрафує великі ваги моделі, зменшуючи ризик перенавчання та підвищуючи стабільність рішення.
Використані параметри: tol=1e-2 – критерій зупинки оптимізації (поріг мінімального покращення); solver=‘sag’ – метод Stochastic Average Gradient, що ефективний для задач з L2-регуляризацією.

Cohen’s Kappa показує ступінь узгодженості між прогнозом та реальними мітками понад рівень випадкового вгадування. Значення 0.933 свідчить про дуже добру узгодженість. ВИСНОВКИ
У ході виконання лабораторної роботи були досліджені різні методи класифікації та проведено їх порівняння на двох датасетах: Census Income та Iris. До основних результатів роботи відносяться такі:
LinearSVC ефективно вирішує задачу бінарної класифікації на датасеті Census Income, показуючи accuracy понад 82% за прийнятний час навчання навіть на великих обсягах даних.
Серед нелінійних ядер SVM найкращий результат на Census Income показало RBF-ядро. Поліноміальне ядро (degree=8) є ризикованим з точки зору перенавчання, а сигмоїдальне ядро виявилось найменш ефективним.
На датасеті Iris SVM (SVC) показав найвищу точність (~98% за крос-валідацією) серед шести порівнюваних алгоритмів. LDA продемонструвала відмінний і стабільний результат при мінімальних обчислювальних витратах.
На датасеті Census Income SVM та Logistic Regression є найкращим вибором: SVM – за якістю класифікації, LR – за балансом між якістю та швидкістю навчання.
RidgeClassifier є ефективним лінійним класифікатором для задач з лінійно роздільними даними. Метрики Cohen’s Kappa та MCC надають більш повну характеристику якості класифікатора порівняно з одним показником accuracy.
Правильна попередня обробка даних (масштабування, обробка пропусків, one-hot кодування) є критично важливим етапом для досягнення якісних результатів будь-яким алгоритмом машинного навчання.
