## Завдання 2.1. Випадкові та гранично випадкові ліси

**Файл коду:** `LR_5_task_1.py`  
**Вхідні дані:** `data_random_forests.txt`  
**Запуски:**
- `python LR_5_task_1.py --classifier-type rf`
- `python LR_5_task_1.py --classifier-type erf`

### Що зроблено
1. Завантажено дані з 3 класами (`Class-0`, `Class-1`, `Class-2`), побудовано вхідний scatter-графік.
2. Дані поділено на train/test (`test_size=0.25`, `stratify=y`).
3. Побудовано 2 моделі:
   - `RandomForestClassifier` (rf),
   - `ExtraTreesClassifier` (erf).
4. Для кожної моделі:
   - побудовано границі класифікації на train/test;
   - розраховано `classification_report`;
   - обчислено ймовірності належності (confidence) для контрольних точок.

### Ключові результати
- **RF test accuracy:** `0.86`
- **ERF test accuracy:** `0.88`
- На тесті ERF дав трохи вищі `precision/recall/f1` у середньому.

### Збережені ілюстрації
Папка `outputs_task1`:
- `rf_input_data.png`, `rf_train_boundary.png`, `rf_test_boundary.png`, `rf_custom_points.png`
- `erf_input_data.png`, `erf_train_boundary.png`, `erf_test_boundary.png`, `erf_custom_points.png`

### Висновок до 2.1
Ансамблеві методи добре працюють на перекривних класах. У цьому наборі даних `ExtraTreesClassifier` показав трохи кращу узагальнювальну здатність на тесті (вищу точність і стабільнішу поведінку меж класифікації), тоді як `RandomForestClassifier` також дав сильний результат, але трохи нижчий за метриками.

## Завдання 2.2. Обробка дисбалансу класів

**Файл коду:** `LR_5_task_2.py`  
**Вхідні дані:** `data_imbalance.txt`  
**Запуски:**
- без балансування: `python LR_5_task_2.py --ignore`
- з балансуванням: `python LR_5_task_2.py --balance --ignore`

### Що зроблено
1. Завантажено двокласовий датасет із вираженим дисбалансом.
2. Побудовано baseline-модель `ExtraTreesClassifier` без `class_weight`.
3. Побудовано balanced-модель `ExtraTreesClassifier` з `class_weight='balanced'`.
4. Порівняно:
   - `classification_report`,
   - `balanced_accuracy`,
   - `weighted F1`,
   - `confusion_matrix`.

### Ключові результати
- **Без балансування:**
  - `balanced_accuracy = 0.6405`
  - на мінорному класі (`Class-0`) recall = `0.32`
- **З балансуванням:**
  - `balanced_accuracy = 0.8399`
  - на мінорному класі (`Class-0`) recall = `0.92`
  - загальна accuracy зменшилась (очікувано), але модель значно краще розпізнає рідкісний клас.

### Збережені ілюстрації
Папка `outputs_task2`:
- `unbalanced/input_data.png`, `unbalanced/test_boundary.png`
- `balanced/input_data.png`, `balanced/test_boundary.png`

### Висновок до 2.2
За наявності дисбалансу класів орієнтація лише на загальну accuracy є хибною. Після увімкнення `class_weight='balanced'` модель суттєво покращила виявлення мінорного класу (ріст recall), що для практичних задач зазвичай важливіше, ніж максимальна загальна точність.

## Завдання 2.3. Пошук оптимальних параметрів (Grid Search)

**Файл коду:** `LR_5_task_3.py`  
**Вхідні дані:** `data_random_forests.txt`  
**Запуск:** `python LR_5_task_3.py`

### Що зроблено
1. Вибрано `RandomForestClassifier`.
2. Задано сітку параметрів:
   - `n_estimators = [50, 100, 200, 300]`
   - `max_depth = [3, 4, 6, 8, None]`
   - `min_samples_split = [2, 4, 8]`
3. Проведено `GridSearchCV` окремо для метрик:
   - `precision_macro`
   - `recall_macro`
4. Для кожного випадку виведено:
   - найкращі параметри;
   - найкращий CV score;
   - тестовий звіт класифікації;
   - top-10 комбінацій параметрів.

### Ключові результати
- Для обох метрик найкраща конфігурація:
  - `{'max_depth': 4, 'min_samples_split': 2, 'n_estimators': 300}`
- `best CV precision_macro = 0.8393`
- `best CV recall_macro = 0.8370`

### Висновок до 2.3
Сітковий пошук дозволив формально підтвердити оптимальний набір гіперпараметрів і уникнути ручного перебору. У поточному наборі даних оптимум для `precision_macro` і `recall_macro` майже збігся, що свідчить про стабільність моделі на різних критеріях якості.

## Завдання 2.4. Відносна важливість ознак (AdaBoostRegressor)

**Дані:** `fetch_california_housing()` із `scikit-learn`  
**Запуск:** `python LR_5_task_4.py`

### Що зроблено
1. Завантажено дані California Housing.
2. Побудовано регресор:
   - базова модель: `DecisionTreeRegressor(max_depth=4)`
   - ансамбль: `AdaBoostRegressor(n_estimators=500, learning_rate=0.05)`
3. Оцінено якість (`MAE`, `RMSE`, `R2`).

- `RMSE = 0.7308`
- `R2 = 0.5993`

### Збережені ілюстрації

### Висновок до 2.4
Найбільший вклад у прогноз вартості житла дає медіанний дохід району (`MedInc`). Географічні ознаки (`Latitude`, `Longitude`) також мають високий вплив. Ознаки на кшталт `AveBedrms` у цьому ансамблі мають меншу цінність і можуть розглядатися як кандидати на відсів у спрощених моделях.

## Завдання 2.5. Прогноз інтенсивності руху (ExtraTreesRegressor)

**Файл коду:** `LR_5_task_5.py`  
**Вхідні дані:** `traffic_data.txt`  
**Запуск:** `python LR_5_task_5.py`

### Що зроблено
1. Завантажено рядкові дані (день, час, суперник, матч yes/no, ціль — кількість авто).
2. Закодовано всі категоріальні ознаки через окремі `LabelEncoder`.
3. Дані поділено на train/test.
4. Навчено `ExtraTreesRegressor`.
5. Обчислено метрики на тесті:
   - `MAE`, `RMSE`, `R2`.
6. Виконано прогноз для невідомої точки:
   - `["Tuesday", "13:35", "San Francisco", "yes"]`

### Ключові результати
- `MAE = 5.1215`
- `RMSE = 7.4212`
- `R2 = 0.7232`
- Прогноз для тестової точки: **`34.85`** авто.

### Висновок до 2.5
Модель на основі гранично випадкових лісів забезпечила якісний прогноз трафіку (`R2 > 0.72`) та добре працює з нелінійними залежностями між категоріальними ознаками і цільовою змінною. Підхід з окремими кодувальниками для кожної ознаки коректно готує дані до регресії та дозволяє робити прогнози для нових записів.

## Підсумковий загальний висновок

У лабораторній роботі підтверджено, що ансамблеві методи:
- підвищують стійкість класифікації на складних/перекривних даних;
- суттєво виграють від коректного врахування дисбалансу класів;
- потребують системного добору гіперпараметрів (Grid Search);
- дають інтерпретованість через оцінку важливості ознак;
- ефективні як у задачах класифікації, так і регресії (прогноз трафіку).
